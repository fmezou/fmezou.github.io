support.progressbar – Progress Bar Widgets¶

This module defines functions and classes to offer progress bar widgets.

To date, the module provides text only widgets for the console.

Using TextProgressBar...

After created a class instance, you must call the compute method to print the progress bar on the standard output (sys.stdout) upon every change of the received length or the content length (typically after a reading of a data block from a stream) as shown in the below example. When the task completed, you must call the finish method which return a string summarizing the amount of computed data.

Example of fetching a file
import urllib.request
import progressbar

url  = "https://docs.python.org" \
       "/3/archives/python-3.5.1-docs-text.zip"
with urllib.request.urlopen(url) as stream:
    content_length = int(stream.info()["Content-Length"])
    length = 0
    progress_bar = progressbar.TextProgressBar(content_length)
    progress_bar.compute(length, content_length)
    while True:
        data = stream.read(1024)
        if not data:
            break
        length += len(data)
        progress_bar.compute(length, content_length)
    progress_bar.finish()
compute(length, content_length=-1)¶

Compute the progress bar and print it.

The printed string have the following content:

aaa% [==============================] lll,lll,lll,lll - rrrr XB/s - eta nn:nn:nn
where
  • aaa: is the percentage of progress
  • ===: is the bar graph of progress (step=1/30)
  • lll,lll,lll,lll: is the number of received bytes
  • rrrr: is the bytes rate where X represent a multiple of the unit bytes per second (G, M or k).
  • nn:nn:nn: is the estimated time to achieve the download.

Note

to prevent the display flicking, the refresh period is limited to 4 times per second.

Parameters:
  • length (int) – A positive integer specifying the length of received data.
  • content_length (int) – A positive integer specifying the content length that the progress bar is going to represent. A negative number specify that the content length is unknown.
Raises:

TypeError – Parameters type mismatch.
finish()¶

Compute the progress bar completion and return the computed string.

The returned string have the following content:

llll XB received - rrrr XB/s in nn:nn:nn
where
  • lll XB: is the number of received bytes where X represent a multiple of the unit byte(G, M or k).
  • rrrr: is the bytes rate where X represent a multiple of the unit bytes per second (G, M or k).
  • nn:nn:nn: is the time of downloading.
Returns:A string representing the progress information.
Return type:str

support.progressbar.isu_format_prefix(value, unit)¶

Return a string using standard prefix for unit.

The returned string represent the value in a compliant International System of Units format, which means with a prefix for unit.
